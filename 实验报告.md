### 仓颉编程任务

本项目为 **HTTP 静态文件服务器**，从 Python 开源项目 "A Simple Web Server"（来自《500 Lines or Less》一书）移植到仓颉语言。

**主要功能：**
- HTTP/1.1 静态文件服务，监听 8080 端口
- 支持 HTML、PNG、JPG 等文件类型的 MIME 识别
- 自动返回 200/404/403 状态码
- **路径穿越攻击防护**（Path Traversal Protection）：实现 `normalizePath()` 路径规范化算法，防止 `../` 攻击
- favicon.ico 静默处理（返回 204 No Content）
- 9 个单元测试用例，覆盖文件服务和路径安全

**项目结构：**
```
cangjie/
├── src/
│   ├── main.cj              # 服务器主程序（约 177 行）
│   └── test/
│       └── server_test.cj   # 单元测试（9 个测试用例）
├── htdocs/
│   └── index.html           # 静态页面根目录
├── screenshots/             # 测试结果截图
├── cjpm.toml                # 项目配置文件
├── README.md                # 项目说明文档
└── 实验报告.md              # 本实验报告
```

**运行方式：**
```bash
cjpm run        # 启动服务器，访问 http://localhost:8080
cjpm test       # 运行单元测试（9/9 PASSED）
```

## 心得体会
- **刘尹媛**：  
    本次项目是将 Python 的 "A Simple Web Server" 移植到仓颉语言的实践。在移植过程中，遇到的主要困难包括：仓颉语言的 TCP Socket API 与 Python 差异较大，需要显式调用 `bind()` 方法；`ArrayList` 的 API 与预期不同（缺少 `remove`、`append`、`insert` 等方法），最终改用 `Array<String>` 配合手动索引管理来实现路径规范化；浏览器会自动规范化 URL 中的 `..`，需要使用 `curl --path-as-is` 才能正确测试路径穿越防护功能。通过这些问题的解决，我对仓颉语言的内存管理、网络编程和文件系统操作有了更深入的理解。仓颉语言作为一门新兴语言，语法清晰严谨，但在标准库的完善程度上还有提升空间。这次项目让我体会到跨语言移植的核心挑战在于理解两种语言设计理念的差异，而非简单的语法翻译。
